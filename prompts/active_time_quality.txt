# Active Time Quality Classification
# Classifies each user message into a mode showing how Copilot contributed.
# Used by the "How I Collaborated" section in the report.
#
# Two detection layers:
# 1. HAND-HOLDING: user correcting, redirecting, or Copilot producing errors
# 2. MODE: based on intent classification of the message content

[hand_holding_user_patterns]
# User correcting or redirecting Copilot (regex, case-insensitive)
\b(no|wrong|not what i|that.s not|didn.t work|try again|still broken|not right|instead|i said|i meant|i asked|that.s incorrect|you missed|go back|undo|revert|abort|stop|don.t|shouldn.t|not impressed|didn.t like|not what|why did you|the issue is|the problem is|that broke|you broke|can you fix|fix this|fix that|fix the|still not|i already told|read (my|the) (previous|last)|look at what i said)\b

[hand_holding_tool_patterns]
# Error signals in tool output (regex, case-insensitive)
\b(error|fail|exception|traceback|broken|crash|UnicodeError|SyntaxError|KeyError|TypeError|ModuleNotFound|ImportError|rejected|denied|exited with exit code [1-9])\b

[modes]
# mode_name | intent_match | description
# First matching rule wins. Course-correcting is checked before these rules.
# Learning BEFORE Researching — "how do I X" / "help me understand" / "what's
# the best way" type queries match both Learning and Researching intents,
# but they are fundamentally knowledge-acquisition questions and should
# count as Learning. Researching stays for fact-finding (cost, limits,
# what options exist, compare A vs B).
# Refining BEFORE Building — a message with both Iterating+Building intents
# is refinement (e.g. "also add X" matches both patterns).
Designing           | Designing,Planning      | Design, strategy, architecture
Analyzing           | Analyzing               | Data analysis, metrics, interpretation
Reviewing           | Reviewing               | Code review, auditing, feedback
Learning            | Learning                | Understanding concepts, knowledge transfer
Researching         | Researching,Investigating,Navigating | Exploring options, investigating
Refining            | Iterating               | Iterating, polishing, improving
Building            | Building                | Writing code, generating files, running commands
Delegating          | Shipping,Configuring    | Git ops, config, installs, routine tasks
